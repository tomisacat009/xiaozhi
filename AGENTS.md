# Xiaozhi Codex Agent Guide

## 目标

- 项目名统一使用：`小智 / Xiaozhi`
- 产品定位：面向高中学习的知识可视化与交互式讲解网站
- 研发目标：持续把知识点做成适合电脑端和手机端访问的正式生产内容

## 协作语言

- 与用户沟通、提交说明、文档撰写优先使用中文
- 代码中的命名、类型、组件名保持英文或现有工程风格

## 当前工程边界

- 正式前端工程基于 `Next.js + TypeScript + React`
- 旧原型目录已经删除，不要再依赖或恢复 `legacy/`
- `docs/superpowers/` 仅作为本地临时目录使用，不纳入 git 跟踪
- `superpower` 相关输出不作为仓库正式文档资产

## 目录约定

- `app/`：App Router 页面入口
- `components/`：通用组件与 demo 外壳组件
- `content/`：学科、模块、知识点、学习路径、demo 注册内容
- `engine/`：可视化 demo 引擎与渲染器
- `features/`：页面级组装逻辑
- `lib/`：加载器、类型、路由与基础工具
- `tests/`：单元测试、集成测试、端到端测试
- `docs/superpowers/`：本地临时 Agent 过程文件，默认忽略

## 开发原则

- 优先沿用现有内容模型和组件边界，不随意新增并行体系
- 新增或重构可视化时，先考虑是否能复用 `engine/renderers/` 里的正式渲染器
- 正文内容必须是可交付状态，不保留“迁移说明”“交互演示迁移线索”“后续重构方向”这类过程性段落
- 页面和交互必须同时考虑桌面端与手机端兼容
- 不把旧方案简单搬回仓库，优先做正式化重写

## 编辑约束

- 修改文件时优先小步、可验证、低破坏
- 不要回退用户已有改动，除非用户明确要求
- 手工编辑代码时使用补丁式修改，保持 diff 清晰
- 删除目录或大文件前，先确认这是用户明确要求的动作

## 验证要求

在声称完成前，优先按改动范围执行相关验证：

- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- 如涉及生产构建：`pnpm exec next build --webpack`

## 提交前检查

- 确认目录结构没有遗留根目录过程文件
- 确认没有把 `docs/superpowers/` 里的临时文件带入提交
- 确认没有把 `.next/`、测试产物或无关缓存当成正式改动
- 确认 README、测试和实现描述一致
